# 14 – Handover-Checkliste

## Onboarding in 10 Minuten
- [ ] `DOC/intern/00_quickstart.md` lesen.
- [ ] App lokal starten (`composer run dev`).
- [ ] Login öffnen und Dashboard prüfen.

## Onboarding in 30 Minuten
- [ ] Kernrouten in `routes/web.php` durchgehen.
- [ ] Modelle + Migrationen (`app/Models`, `database/migrations`) verstehen.
- [ ] Kerncontroller lesen (`Customer`, `Device`, `Inspection`, `Import`, `Report`).
- [ ] Einmal manuell End-to-End-Flow testen.

## Onboarding in 120 Minuten
- [ ] Erste kleine Änderung implementieren (z. B. zusätzliches Feld in Device-Form).
- [ ] Migration + Validierung + View + Controller konsistent anpassen.
- [ ] Tests ergänzen und lokal ausführen.

## Definition of Done (Feature)
- [ ] Fachliche Anforderung im Code umgesetzt.
- [ ] Validierung/Fehlerfälle berücksichtigt.
- [ ] Migrationen + Modell + UI + Controller konsistent.
- [ ] Tests ergänzt oder bestehende Tests angepasst.
- [ ] Doku in `DOC/intern/` aktualisiert, wenn Verhalten geändert wurde.

- Im Repository keine verlässliche Owner-/Teamliste vorhanden -> `UNBEKANNT / FEHLT`.
